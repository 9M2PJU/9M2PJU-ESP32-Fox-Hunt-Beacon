# 9M2PJU ESP32 Fox Hunt Beacon — Project Notes

## Project Overview
ESP32 firmware for amateur radio fox hunting beacon. ESP32 handles timing,
CW ID, warble tones, PTT control, battery monitoring, and web admin UI.
The actual RF signal is generated by an external handheld radio keyed via PTT.

- Repository: github.com:9M2PJU/9M2PJU-ESP32-Fox-Hunt-Beacon.git
- License: GPLv3-or-later
- Framework: PlatformIO + Arduino ESP32 core

## Build & Tooling
- PlatformIO installed at `~/.platformio/penv/bin/pio` (not on PATH)
- Build: `pio run -e esp32dev` (or any of 32 board environments)
- Default env: `esp32dev`
- Platform: espressif32@^6.7.0
- Pre-build script: `tools/compress.py` (gzip-compresses web UI assets)

## Key Libraries
- ESPAsyncWebServer (me-no-dev @ ^3.6.0) — async non-blocking web server
- AsyncTCP (me-no-dev @ ^3.3.2) — async TCP for ESPAsyncWebServer
- OneButton (mathertel @ 2.6.2) — debounced multi-click button handling
- U8g2 (@ ^2.28.8) — OLED SSD1306 display driver
- TFT_eSPI (@ ^2.5.0) — TFT ST7789/ST7735 display driver
- GxEPD2 (@ ^1.5.6) — E-Ink SSD1680 display driver

## Display System (DISPLAY_TYPE compile-time flag)
- 0 = no display (default, web admin only)
- 1 = OLED SSD1306 via U8g2 (I2C)
- 2 = TFT via TFT_eSPI (SPI, ST7789/ST7735)
- 3 = E-Ink SSD1680 via GxEPD2 (SPI, 250x122 or 296x128)

Display functions: displayInit, displayStartupScreen, displayUpdate,
displayMenu, displayAPMode, displayPower, displayClear

## Web Admin UI
- ESPAsyncWebServer on port 80 (non-blocking)
- WiFi AP mode with captive portal DNS redirect
- Web assets (HTML/CSS/JS) in data_embed/, gzip-compressed at build time
  by tools/compress.py, embedded as PROGMEM arrays in include/web_assets.h
- AP SSID: FoxBeacon-XXXX (last 4 hex of MAC)
- AP auto-off: configurable timeout (default 10 min, 0=never)
- Activity tracking: resets on client connection or HTTP request
- Config API: GET /api/config (JSON), POST /api/config (form data)
- Commands: POST /api/test, /api/ptt_test, /api/defaults, /api/reboot
- Branding: "9M2PJU Fox Hunt Beacon" (title + h1 header)

## Button Handling (OneButton library)
- Single click: menu down / test TX on status / wake display
- Double click: toggle menu item / open menu / wake display
- Long press (2s): exit menu to status screen
- BUTTON_PIN defined per board in platformio.ini (usually GPIO 0)

## Display Eco Mode
- Configurable on/off via config.displayEcoMode
- Timeout: 15 seconds (DISPLAY_ECO_TIMEOUT_MS)
- Grace period: 10 seconds after boot (DISPLAY_ECO_GRACE_MS)
- Only activates on status screen (not in menu)
- Wakes on any button press

## Config Storage
- Binary via Arduino Preferences (NVS flash partition)
- Key prefix: "foxbeacon" namespace
- Fields: callSign, foxId, beaconMode, foxSyncEnabled, startupDelaySeconds,
  transmitSeconds, idleSeconds, beaconIdIntervalSeconds, cwWpm, cwToneHz,
  warbleEnabled, warbleLowHz, warbleHighHz, warbleStepMs, pttActiveLow,
  pttLeadMs, pttTailMs, batteryEnabled, batteryScale, lowBatteryVoltage,
  wifiApEnabled, wifiApTimeoutMinutes, displayEcoMode

## Serial Commands (115200 baud)
- show / help / ? — print config + command list
- test — queue test transmission
- ptt_test — key PTT for 1.2s (no audio)
- defaults — restore compile-time defaults
- reboot — restart ESP32
- set <name> <value> — set any config field
  - call, fox, mode, fox_sync, startup, tx, idle, beacon_id
  - wpm, tone, warble, warble_low, warble_high, warble_step
  - lead, tail, ptt, battery, battery_scale, low_battery
  - wifi_ap, wifi_ap_timeout, eco_mode

## On-Screen Menu (display boards only)
- Double-click button to open menu
- Single click: navigate down
- Double click: toggle selected item
- Long press (2s): exit menu
- Menu auto-exits after 30s inactivity
- Items: WiFi AP, Warble, Fox Sync, Battery, Beacon Mode, Eco Mode, Exit

## Board Environments (32 total)
- 5 generic ESP32 boards (esp32dev, doit, lolin32, s3-devkit, c3-devkit)
- 9 Heltec OLED boards (wifi-kit-32, wifi-lora-32, wireless-stick variants)
- 2 Heltec TFT boards (wireless-tracker, vision-master-t190)
- 3 Heltec E-Ink boards (wireless-paper, vision-master-e213, vision-master-e290)
- 1 Heltec no-display board (capsule-sensor-v3)
- 4 TTGO/LilyGO OLED boards (lora32, t-beam variants)
- 3 TTGO/LilyGO TFT boards (t-display, t-display-s3, t-watch)
- 5 TTGO/LilyGO no-display boards (t3-s3, t-oi-plus, t1, t7 variants)

## File Structure
- include/beacon_config.h — config struct + defaults + firmware version
- include/display.h, display_config.h — display API + config
- include/web_admin.h — web admin API
- include/web_assets.h — AUTO-GENERATED by tools/compress.py (gitignored)
- src/main.cpp — main firmware (state machine, serial, button, display)
- src/display.cpp — display backends (OLED/TFT/E-Ink)
- src/web_admin.cpp — async web server + WiFi AP
- data_embed/index.html, style.css, script.js — web UI source files
- tools/compress.py — pre-build gzip compression script
- platformio.ini — 32 board environments + shared sections
- docs/ — user documentation (installation, configuration, troubleshooting, etc.)
- images/fox-beacon-logo.png — branding logo
